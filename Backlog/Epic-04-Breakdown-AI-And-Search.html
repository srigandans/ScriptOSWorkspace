#164 Epic Breakdown, AI & Search
Sprint 6 Alpha Milestone ● Priority: Critical breakdown ai search alpha wga
Description
Deliver three services that transform ScriptOS into an intelligent production platform: (1) the Breakdown Service, which auto-detects production elements (characters, locations, props, wardrobe, VFX, stunts) from the Script AST using NLP within 30 seconds of every checkpoint event; (2) the AI Service, which provides WGA Article 72-compliant dialogue suggestions, format corrections, and consistency checks with full ledger tracking, company consent gating, Anthropic prompt caching, and org-level token quotas; and (3) the Search Service, which indexes all script content in Elasticsearch 8.9+ for sub-500ms hybrid BM25 + vector search with native RRF.

This epic concludes at the Alpha milestone (Week 14): 10 external writers invited to use the editor, Series Bible, and AI suggestions.
Reference: Wiki/15-roadmap.md Sprint 6, Wiki/05-ai-governance.md, Wiki/08-production-modules.md.
Acceptance Criteria (Epic-level Definition of Done)
  • Breakdown elements auto-detected for 5 test scenes with ≥ 80% F1 vs hand-tagged ground truth
  • Breakdown re-triggered within 30s of script.checkpointed NATS event for changed scenes
  • AI dialogue_suggestion end-to-end: consent → context assembly → Anthropic API → character guard → ledger write → suggestions displayed in UI
  • AI ledger DELETE trigger verified: deletion attempt raises DB exception; entries are permanent
  • Professional org quota of 500K tokens enforced; 429 returned with Retry-After on exhaustion
  • Search results returned within 500ms (P95) for a 100-scene test script on staging
  • Breakdown Service, AI Service, Search Service all deployed to staging with health checks passing
  • Alpha milestone: 10 external writers onboarded with editor + bible + AI access
#165 Feature Breakdown Service — NLP Element Detection and Production Catalog
Sprint 6 ● Priority: Critical breakdown nlp grpc
Auto-detect production elements from Script AST checkpoint events and expose a version-locked catalog. NLP pipeline identifies characters (matched against Bible stubs), locations (parsed from scene headings), and six other categories using NER and keyword matching. Each detection carries a confidence score. The catalog is version-controlled and immutable after publish-saga lock.
#168 User Story As a producer I can see auto-detected breakdown elements for every scene
8 pts Sprint 6 ● Critical
Manual breakdown entry takes 2–4 days for a feature film. Auto-detection reduces that to hours. The breakdown panel groups elements by category with confidence badges for rapid human verification. Version-locking after publish ensures the breakdown used for scheduling is stable.
Acceptance Criteria
  • Breakdown panel shows auto-detected elements per scene grouped by category (Cast, Locations, Props, Wardrobe, VFX, Stunts)
  • HIGH badge (≥ 0.9) green; MEDIUM (0.7–0.9) amber; LOW (< 0.7) red
  • Elements refresh within 30s after a checkpoint event
  • After publish lock: elements show padlock; add/edit/delete controls hidden
  • F1 ≥ 80% on 5 test scenes vs hand-tagged ground truth

#169 User Story As a 1st AD I can manually add, edit, and remove breakdown elements to correct auto-detections
5 pts Sprint 6 ● High
NLP is imperfect. A 1st AD must correct missed elements, edit mis-categorised detections, and delete false positives. Manual edits survive re-detection runs — is_manual = true elements are never overwritten by the NLP pipeline.
Acceptance Criteria
  • Manual add: creates element with is_manual = true, confidence = 1.0; survives NLP re-detection
  • Edit: saves immediately with optimistic UI update; rollback on gRPC error
  • Delete: removes after confirmation; undo snackbar for 5 seconds
  • Version-locked elements: mutation returns 409; UI disables all controls
  • Manual elements show pen icon to distinguish from NLP detections

#166 Feature AI Service — WGA-Compliant Dialogue Suggestions and AI Governance
Sprint 6 ● Priority: Critical ai wga consent ledger anthropic
Build the AI Service with WGA Article 72 compliance at its core: company consent model, per-feature writer opt-in, context assembler with prompt caching, Claude Haiku/Sonnet/Opus model router, Anthropic SDK integration, immutable AI ledger with DB trigger, dialogue_suggestion and format_correction features, and Redis org token quota + per-user soft limits.
#170 User Story As a writer I can request AI dialogue suggestions with WGA-compliant consent and full ledger tracking
8 pts Sprint 6 ● Critical
WGA Article 72 compliance is non-negotiable. Company must enable AI features. Writer must opt in per feature type. Every request and writer action is logged in the immutable AI ledger with prompt hash and modification delta. Character voice forbidden_words are checked before returning any suggestion.
Acceptance Criteria
  • First use: consent banner shown; toggle stores ai.feature_consents row
  • Cmd+Shift+A triggers: consent check → context assembly → Claude Sonnet call → character guard → ledger write → 2–3 suggestions displayed
  • Accept: writer_action = 'accepted', modification_delta = '' logged to ledger
  • Modify: writer_action = 'modified', modification_delta = character diff logged
  • Reject: writer_action = 'rejected' logged
  • Forbidden word in suggestion: suggestion withheld; warning shown inline
  • Ledger entry not deletable: DB trigger raises exception on DELETE attempt

#172 User Story As a writer I can search all script content and receive relevance-ranked results in under 500ms using hybrid keyword and semantic search
5 pts Sprint 6 ● High
Writers navigate large multi-episode series by intent ("the scene where Walter first sees the RV"), not just keywords. Hybrid BM25 + semantic search powered by Elasticsearch RRF gives far better recall. The Cmd+K command palette keeps search instantly accessible.
Acceptance Criteria
  • Cmd+K opens search palette from anywhere in the web app
  • Results returned in < 500ms (P95) for 100-scene script on staging hardware
  • Each result shows: scene number badge, heading, matched excerpt with highlighted terms
  • Clicking a result navigates editor to that scene, scrolled into view
  • Semantic query without exact keywords returns correct scene in top 3 (human eval)
  • Index updated within 60s of any checkpoint; new/edited scenes immediately searchable
