#230EpicRevision, Watermark & On-Set Supervisor
Sprints 9–10 ● Priority: Critical revisionwatermarksupervisortauripowersync
Description
Deliver four capabilities that bridge script authoring and physical production: (1) Revision Color Workflow — 13-colour WGA sequence state machine with locked version management, scene-level diff, and per-recipient watermarked distribution saga; (2) Watermark Service — steganographic embedding in PDF (character spacing) and FDX (zero-width characters); (3) Call Sheet Saga — Temporal workflow generating industry-standard call sheet PDF with AD approval gate and crew distribution; (4) Supervisor Module + PowerSync — Tauri 2.x on-set app with SQLite + SQLCipher encryption, PowerSync offline sync, take logging, and continuity assertions. Reference: Wiki/15-roadmap.md Sprints 9–10, Wiki/06-offline-sync.md, Wiki/11-security-compliance.md.
Acceptance Criteria
  • Revision color state machine: 13 colours enforced; locked pages reject colour change
  • Watermarked PDF round-trip: embed recipient ID → extract from leaked copy → recover correct ID (confidence > 0.95)
  • Revision distribution saga: correct watermarked PDF delivered to each recipient
  • Call sheet PDF generated and distributed to 3 test crew members
  • Tauri app runs on macOS; script supervisor logs takes offline for 1 hour; all sync on reconnect
  • SQLCipher encryption verified: DB file unreadable without key (hex dump test)
  • PowerSync sync works for supervisor and AD roles with correct data isolation
  • Supervisor Service deployed to staging with passing integration tests
